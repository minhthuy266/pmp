## Corrections and Clarifications

- Process Groups are not project phases and can repeat within a project or phase.
- ITTO counts should not be memorized as isolated facts; understand purpose and information flow.
- The Performance Measurement Baseline integrates scope, schedule, and cost baselines. Treating it
  as an unrelated fourth baseline can obscure that relationship.
- PMIS is broader than one software application.
- Autocratic decision-making is not automatically wrong when authority and context support it.
- Meeting documentation should be tailored; not every meeting needs detailed minutes.
- A change request is evaluated according to the Change Management Plan and delegated authority;
  the CCB does not necessarily decide every minor change.
- Updating a controlled plan component or baseline still requires the applicable change-control
  process.

## Source Priority

Andrew Ramdayal's slides are used as a learning and coverage source. Current PMI terminology,
information flow, governance, and context-sensitive reasoning take priority over fixed counts or
